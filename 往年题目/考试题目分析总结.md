# COMP6080 往年考试题目分析总结

## 总体结构

所有年份的考试都遵循相似的结构：
- **导航组件**：Header/Navbar/Sidebar（不同年份位置不同）
- **Dashboard页面**：显示游戏统计（获胜数/剩余数）
- **3个游戏页面**：不同年份的游戏类型不同
- **纯前端实现**：使用localStorage存储状态，无后端依赖

---

## 20T3 考试

### 考试题目
- **导航**：顶部Header（80px高，包含logo和导航链接）
- **Footer**：底部固定（50px高，背景色#999999）
- **Dashboard**：显示"Games won: X (reset)"，从API获取初始值
- **游戏1 - Blanko**：12个盒子，显示字符串，3个随机位置输入框

### 对应文件位置
- **原始题目**：`20T3-exam/exam-sample/README.md`（但这是Create React App的默认README）
- **实际题目**：根据之前创建的`20T3-exam-frontend/`项目结构
- **实现文件**：

## 游戏类型总结

### 重复出现的游戏类型：
1. **Blanko**：20T3, 23T3（字符串填空游戏）
2. **Slido**：20T3, 23T3（滑动拼图游戏）
3. **Tetro**：20T3, 23T3（简化版俄罗斯方块）
4. **Operations**：22T1, 24T1（操作符填空游戏）
5. **Memory**：21T3, 22T1, 24T1（记忆游戏，但规则略有不同）
6. **Space**：22T1, 24T1（太空入侵者游戏）

### 独特的游戏类型：
1. **Math**：21T3（随机运算符游戏）
2. **Connect 4**：21T3（10x10棋盘四子棋）
3. **Number Memory**：25T1（数字记忆游戏）
4. **Treasure Hunt**：25T1（寻宝游戏）
5. **Flappy Bird**：25T1（简化版Flappy Bird）

## 技术栈

所有年份都使用：
- **React + Vite**（或Create React App）
- **React Router**（路由管理）
- **localStorage**（状态持久化）
- **纯前端实现**（无后端依赖）

---

## 文件结构模式

所有年份的项目都遵循相似的文件结构：
```
{年份}-exam-frontend/
├── src/
│   ├── components/       # 导航组件
│   ├── pages/            # 页面组件（Dashboard + 3个游戏）
│   ├── utils/            # 工具函数（API调用、localStorage）
│   ├── data/             # 数据文件（部分年份需要）
│   ├── App.jsx
│   └── main.jsx
├── package.json
├── vite.config.js
└── index.html
```
