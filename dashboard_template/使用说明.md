# Dashboard Template 使用说明

## ✅ 模板已包含响应式设计

这个模板已经融合了完整的响应式设计，可以直接复制使用。

## 📁 文件清单

```
dashboard_template/
├── App.js          # 主应用文件（包含Header、Footer、路由）
├── App.css         # 完整的响应式样式（包含所有断点）
├── Dashboard.jsx   # Dashboard页面（已响应式）
├── dash.jsx        # Dashboard 2x2网格布局（已响应式）
├── Blanko.jsx       # Blanko游戏占位符
└── 使用说明.md      # 本文件
```

## 🚀 快速开始

### 1. 复制文件到你的项目

将以下文件复制到你的React项目：
- `App.js` → `src/App.js`
- `App.css` → `src/App.css`
- `Dashboard.jsx` → `src/Dashboard.jsx`
- `dash.jsx` → `src/dash.jsx`
- `Blanko.jsx` → `src/Blanko.jsx`（可选，如果不需要可以删除）

### 2. 安装依赖

确保你的项目已安装：
```bash
npm install react react-dom react-router-dom
# 或
yarn add react react-dom react-router-dom
```

### 3. 修改logo（可选）

在 `App.js` 中，logo目前使用占位符：
```javascript
const logo = "https://via.placeholder.com/50x50";
```

你可以：
- 替换为你的logo路径：`const logo = "/logo.png";`
- 或使用import：`import logo from "./logo.svg";`

### 4. 自定义路由和页面

根据你的需求修改：
- 路由路径（`/`, `/dash2`, `/blanko` 等）
- 页面组件（Dashboard, Dash, Blanko等）
- 导航链接文字

## 📱 响应式断点

模板包含以下响应式断点：

| 屏幕尺寸 | 断点 | Header高度 | 文字显示 |
|---------|------|-----------|---------|
| 桌面端 | > 1400px | 80px | 完整文字 |
| 平板端 | ≤ 1400px | 70px | 完整文字 |
| 移动端 | ≤ 800px | 60px | 缩写文字 |
| 小屏移动端 | ≤ 480px | 50px | 缩写文字 |

## 🎨 响应式特性

### Header导航
- **桌面端**：显示完整文字（Home | Dash2 | Blanko）
- **移动端**：自动切换为缩写（H | D2 | B）

### 布局适配
- Dashboard：居中布局，字体大小自适应
- Dash（2x2网格）：桌面端横向排列，移动端自动垂直排列
- 所有间距、字体大小都会根据屏幕尺寸自动调整

## ⚙️ 功能说明

### Dashboard页面
- 显示"Games won"计数
- 从API获取初始值（`https://cgi.cse.unsw.edu.au/~cs6080/raw/data/info.json`）
- 使用localStorage持久化存储
- Reset按钮重置计数

### Dash页面
- 2x2网格布局
- 响应式：移动端自动调整为垂直布局
- 显示numWon数据

## 🔧 自定义修改

### 修改导航链接
在 `App.js` 的 `Header` 组件中：
```jsx
<Link to="/your-path">
  <span className="nav-long">完整文字</span>
  <span className="nav-short">缩写</span>
</Link>
```

### 修改颜色和样式
在 `App.css` 中修改：
- Header背景色：`.header { background-color: #eeeeee; }`
- Footer背景色：`.footer { background-color: #999999; }`
- 文字颜色：`.header-nav a { color: #333; }`

### 添加新页面
1. 创建新组件文件（如 `Game1.jsx`）
2. 在 `App.js` 中导入：`import Game1 from './Game1';`
3. 添加路由：`<Route path="/game1" element={<Game1 />} />`
4. 在Header中添加导航链接

## 📝 注意事项

1. **logo路径**：记得替换logo占位符为你的实际logo
2. **API地址**：如果API地址不同，修改 `App.js` 中的URL
3. **路由路径**：根据你的需求修改路由路径
4. **组件命名**：可以根据需要重命名组件
